getReportPurchases(pageSize, pageNumber, sortBy, expand, granularity, dimensions, startDate, endDate, regexFilterEnabled, effectiveStartDate, effectiveEndDate, did, phoneNumberType, startAuthUserId, endAuthUserId, status, orgId)

Get LCR report.

This method queries Druid for a Phone Number Purchase Stats database report. Some Druid terminology is required to use it. Each query parameter besides dimensions, _pgn, and _pgs, is a Druid dimension. From the Druid documentation: Dimensions: Aspects or categories of data, such as languages or locations. For example, with language and country as the type of dimension, values could be "English" or "Mandarin" for language, or "USA" or "China" for country. In Druid, dimensions can serve as filters for narrowing down hits (for example, language = "English" or country = "China"). When using a dimension query parameter with a value the report will return events with granularity of that dimension where the dimension value is equal to the value you specified. If you want to receive events with dimension based granularity with no filter, opt to use the dimensions query parameters, which is a comma separated list of values that use the other query parameters as dimension names. Each parameter needs to be in the screaming snake case format, or words capitalized with underscores between words (i.e. CUSTOMER_ID).
Parameters:
Name Type Description
pageSize integer The total page size requested
pageNumber integer The page number requested
sortBy string variable name requested to sort by
expand array variable name requested by expand list
granularity string The granularity field determines how data gets bucketed across the time dimension, or how it gets aggregated by hour, day, minute, etc. all buckets everything into a single bucket. none does not bucket data (it actually uses the granularity of the index - minimum here is none which means millisecond granularity).
dimensions string Dimensions are really any other parameter specified, excluding granularity. To specify multiple dimensions use comma separated values.
startDate string Start dateTime for ISO-8601 Intervals. This defines the time ranges to run the query over. If not set will always default to now minus 15 days.
endDate string End dateTime for ISO-8601 Intervals. This defines the time ranges to run the query over. If not set will always default to startDate plus 30 days.
regexFilterEnabled boolean Specifies if all dimension filters should be implemented as regexps.
effectiveStartDate string Effective date number service starts
effectiveEndDate string Effective date number service ends
did string Number purchased (Can use with regex for npa, etc.)
phoneNumberType string Type of number purchased (ie. us-domestic, tollfree...) TOLL_FREE, CN_DOMESTIC, US_DOMESTIC,
startAuthUserId string User id for the number purchase
endAuthUserId string User id for the number disconnect order
status string Status of the number purchase COMPLETE, FAILED, BACKORDERED, PENDING, ERROR, DISCONNECTED, DISCONNECT_PENDING, DISCONNECT_FAILED,
orgId string Organization id for the number purchase
